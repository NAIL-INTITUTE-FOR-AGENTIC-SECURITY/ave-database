## ✅ Completed

### Phase 1: Foundation (Complete)
- [x] AVE card schema and taxonomy (14 categories)
- [x] 36 vulnerability cards from 29 experiments
- [x] Python CLI toolkit (list, show, search, stats, validate, submit, export)
- [x] Gamification engine (XP, badges, tiers, Hall of Fame)
- [x] AVSS scoring system (Agentic Vulnerability Scoring System)
- [x] Card validation and schema enforcement
- [x] CC-BY-SA-4.0 licensing

### Phase 2: Community Infrastructure (Complete)
- [x] GitHub Issue templates (AVE submission, private disclosure)
- [x] Pull Request template
- [x] Contributing guide (323 lines — schema, evidence standards, review process)
- [x] Security policy (responsible disclosure, 90-day timeline)
- [x] Code of Conduct
- [x] CI pipeline (card validation on every PR)
- [x] Auto-generated Hall of Fame (GitHub Actions)

### Phase 3: Public Platform (Complete)
- [x] Documentation site (GitHub Pages — 40 HTML pages)
- [x] Searchable card browser with severity/category filters
- [x] Taxonomy visualization page
- [x] Contributor guide page
- [x] Individual card detail pages (36)
- [x] GitHub Discussions (5 categories: Proposals, Research, Defences, CTF, General)
- [x] Public read-only API (FastAPI — 10 endpoints)
- [x] CTF event portal (events, challenges, rules, scoring)
- [x] Published research artifacts (findings, statistics, defence architecture, paper abstract, ebook preview)
- [x] Docs sync and public-repo export pipeline

### Phase 4: Growth Automation (Complete)
- [x] Weekly community digest (auto-generated Monday report)
- [x] New contributor welcome bot
- [x] Auto-labelling for issues (severity, category, triage)
- [x] Stale issue/PR management
- [x] Comprehensive project roadmap

## 📋 Planned

### Phase 6: Industry Adoption

- [ ] **AVE Feed (RSS/Atom)** — Machine-readable vulnerability feed for security tooling
- [ ] **IDE Extensions** — VS Code extension for AVE card lookup during development
- [ ] **CI/CD Integration** — GitHub Action: "Scan my agent code for AVE vulnerabilities"
- [ ] **SBOM Integration** — AVE references in Software Bill of Materials
- [ ] **Partner Programme** — Framework maintainers can link mitigations to AVE cards
- [ ] **Enterprise CTF** — Private red team assessments for enterprise AI deployments

### Phase 7: Research Scaling

- [ ] **Community Experiments** — Open experiment framework for community-run reproductions
- [ ] **Cross-Model Study Portal** — Web interface for browsing cross-model comparison data
- [ ] **Defence Benchmarks** — Standardised benchmark suite for evaluating agentic defences
- [ ] **Automated AVE Discovery** — Research Scout agent scans for new vulnerability classes
- [ ] **Longitudinal Studies** — 90-day agent observation experiments

### Phase 8: Standards & Governance

- [ ] **AVE Advisory Board** — Community-elected board for card review and taxonomy decisions
- [ ] **Formal AVE Standard** — RFC-style specification for the AVE card format
- [ ] **Certification Programme** — "NAIL Certified" badge for agent systems that pass AVE-based testing
- [ ] **Insurance Integration** — AVE-based risk scoring for AI agent insurance policies
- [ ] **Regulatory Engagement** — Feed AVE data into EU AI Act compliance frameworks
