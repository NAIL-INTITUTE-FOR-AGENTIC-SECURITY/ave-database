## Slide 3: The Gap

### Current Security Standards Don't Cover AI Agents

| Standard | Covers AI Agents? | Gap |
|----------|:-:|-----|
| CVE / NVD | ❌ | Software bugs, not AI behaviours |
| CVSS | ❌ | No autonomy or propagation metrics |
| OWASP Top 10 for LLMs | ⚠️ | LLM-specific, not agent-specific |
| MITRE ATT&CK | ⚠️ | Traditional tactics, not agentic |
| CWE | ❌ | Code weaknesses, not emergent behaviour |

**Result**: No common language, no scoring, no standards for
agentic AI security.

## Slide 4: The Solution

### AVE — The CVE for AI Agents

The **Agentic Vulnerability Enumeration** database:

- 📋 **50+ vulnerabilities** across **14 categories**
- 📊 **AVSS scoring** — severity metrics designed for AI agents
- 🗺️ **MITRE ATT&CK mappings** for every entry
- 🔬 **Experimental validation** with controlled PoC tests
- 🆓 **100% open source** — CC BY 4.0

Plus: certification, training, insurance models, regulatory mapping,
and a commercial API platform.
