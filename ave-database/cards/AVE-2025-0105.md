# AVE-2025-0105 — State Persistence Deserialization

| Field | Value |
|-------|-------|
| **AVE ID** | AVE-2025-0105 |
| **Name** | State Persistence Deserialization |
| **Category** | memory |
| **Severity** | critical |
| **Status** | proven |
| **AVSS Score** | 7.4 (critical) |
| **CWE** | CWE-502 (Deserialization of Untrusted Data) |
| **Affected Frameworks** | LangGraph, smolagents |
| **CVEs** | CVE-2025-64439, CVE-2026-27794, CVE-2026-28277 |

---

## Summary

Agentic frameworks that persist agent state (checkpoints, caches, stores) use **unsafe deserialization by default** — including pickle fallbacks, msgpack object reconstruction, and JSON constructor modes — enabling arbitrary code execution when the persistence layer is compromised.

This vulnerability is distinctive because **all three serialization modes** in LangGraph's `JsonPlusSerializer` were independently exploitable, creating a near-total compromise of the persistence layer's trust boundary.

---

## Mechanism

Agent state serializers prioritize flexibility over security:

1. **JSON Constructor Mode** (CVE-2025-64439): The `lc:2` type indicator in LangChain's serialization format triggers constructor-based deserialization. A crafted payload like `{"lc":2, "type":"constructor", "id":["os","system"], "args":["whoami"]}` executes arbitrary system commands when loaded.

2. **Pickle Fallback** (CVE-2026-27794): When `pickle_fallback=True` (the default for caches), any object that fails JSON/msgpack serialization falls back to `pickle.loads()`. An attacker writes a pickle payload to the cache backend, achieving RCE on next read.

3. **Msgpack Unsafe Reconstruction** (CVE-2026-28277): The msgpack ext type handling reconstructs arbitrary Python objects from serialized state, enabling object injection during checkpoint load.

### Attack Flow

```
1. Attacker gains write access to persistence backend
   (Redis, PostgreSQL, filesystem — often lower-privilege than app)
2. Attacker writes crafted serialized payload to checkpoint/cache/store
3. Agent loads checkpoint on next invocation
4. Unsafe deserialization executes attacker payload
5. Full code execution in agent runtime context
```

---

## Blast Radius

- **Arbitrary code execution** in the agent runtime
- **Full environment variable access** (API keys, secrets)
- **Lateral movement** to connected services
- **Data exfiltration** from agent memory/context
- Escalation from "persistence-layer compromise" to "full application compromise"

---

## Defences

### 1. Strict Serialization Mode
Serialize only explicitly allowed types via schema-derived allowlists. Set `LANGGRAPH_STRICT_MSGPACK=true`. Default to `pickle_fallback=False`.

### 2. Persistence Layer Integrity
Apply cryptographic integrity tags (HMAC) to serialized state. Restrict write access to persistence backends using minimal-privilege database roles. Rotate credentials on suspected compromise.

### 3. Safe-by-Default Serialization
Use allowlist-only serialization as the default configuration, requiring explicit opt-out for unsafe modes. Never default to `pickle_fallback=True`.

### 4. Input Validation on Deserialized State
Validate the structure and types of deserialized data before use, rejecting any unexpected types or constructor references.
