# 📊 Slide Deck Templates

> Outlines for NAIL Institute training slide decks.
> Full decks are provided to NAIL Authorised Instructors.

## Deck Structure (All Tracks)

Every slide deck follows this structure:

### Opening (5 slides)
1. **Title Slide** — Track name, instructor, date, NAIL logo
2. **Agenda** — Module overview for the day
3. **Housekeeping** — WiFi, breaks, restrooms, emergency exits
4. **Introductions** — Instructor background, student round-table
5. **Learning Objectives** — What you'll know by the end

### Module Content (varies)
- Maximum 30 slides per 90-minute module
- Every 5th slide should be interactive (question, poll, exercise)
- Use the NAIL visual theme (dark background, green accents)
- Include speaker notes on every content slide

### Closing (3 slides)
1. **Key Takeaways** — 3–5 bullet summary
2. **Resources** — Links, docs, next steps
3. **Feedback** — QR code to feedback survey

## Design Guidelines

### Typography
- **Headings**: Inter Bold, 36pt
- **Body**: Inter Regular, 24pt
- **Code**: JetBrains Mono, 20pt
- **Minimum font size**: 20pt (for readability)

### Colour Palette
| Use | Colour | Hex |
|-----|--------|-----|
| Background | Dark Navy | `#0A1628` |
| Primary Text | White | `#FFFFFF` |
| Accent | NAIL Green | `#00D68F` |
| Warning | Amber | `#FFB800` |
| Critical | Red | `#FF3D71` |
| Code Background | Dark Grey | `#1A2332` |

### Visual Elements
- Use diagrams over bullet points where possible
- Include AVE card examples on every vulnerability slide
- Show attack → defence flow diagrams
- Use before/after code comparisons
- Include real-world incident screenshots (anonymised)

## Module-Specific Outlines

### T1-M1: Introduction to Agentic AI Vulnerabilities (40 slides)
```
Slide 1-5:   Opening
Slide 6-10:  What is Agentic AI? (definitions, examples, growth data)
Slide 11-15: Why Traditional Security Falls Short
Slide 16-20: The AVE Mission (history, community, database stats)
Slide 21-30: Case Studies (3 real incidents with analysis)
Slide 31-35: Interactive: "Is this an agent vulnerability?" quiz
Slide 36-40: Summary and preview of next module
```

### T1-M2: The AVE Taxonomy (60 slides)
```
Slide 1-5:   Overview of 14 categories
Slide 6-10:  Prompt Injection (direct, indirect, examples)
Slide 11-14: Goal Drift (definition, detection, examples)
Slide 15-18: Tool Misuse (abuse patterns, real cases)
Slide 19-22: Identity Spoofing (impersonation, auth bypass)
Slide 23-26: Memory Poisoning (context manipulation, RAG attacks)
Slide 27-30: Privilege Escalation (capability escalation, sandbox escape)
Slide 31-34: Resource Abuse (compute waste, API abuse)
Slide 35-38: Output Manipulation (hallucination steering, format abuse)
Slide 39-42: Coordination Failure (multi-agent, consensus, race conditions)
Slide 43-46: Supply Chain (model, plugin, data supply chain)
Slide 47-50: Data Exfiltration, DoS, Model Extraction, Social Engineering
Slide 51-55: Interactive: Classify 10 scenarios
Slide 56-60: Summary and category cheat sheet
```

### T1-M3: AVSS Scoring System (25 slides)
```
Slide 1-5:   Why scoring matters (prioritisation, communication)
Slide 6-10:  AVSS components breakdown
Slide 11-15: Scoring walkthrough (2 examples, step by step)
Slide 16-20: AVSS vs CVSS comparison
Slide 21-25: Interactive: Score 3 vulnerabilities as a group
```
